# 简答题

## 1）动作识别中的“时空特征”指什么？为何视频分析需要同时建模空间和时间信息？

答案：动作识别中的“时空特征”是指同时包含空间外观信息和时间运动信息的特征表示。空间信息主要描述单帧图像或点云中的人体姿态、物体外观、场景背景、几何结构和空间关系；时间信息则描述连续帧之间的动作变化、姿态转移、运动轨迹和动态演化过程。视频不是孤立图像的集合，而是随时间变化的序列，因此只看空间信息可能只能判断“画面中有什么”，却难以区分“正在做什么”，例如走路和跑步在单帧中可能相似，但在时间变化速度和运动节奏上不同；只看时间信息又可能缺少人体、物体和场景上下文。动作识别需要同时建模空间和时间信息，才能更完整地理解动作的主体、环境和动态过程，提高识别准确率和鲁棒性。

## 2）Two-stream 网络的基本结构是什么？其优势和局限性分别是什么？

答案：Two-stream 网络通常由两个相对独立的卷积神经网络分支组成，即空间流和时间流。空间流以 RGB 图像帧为输入，主要学习人体、物体、场景等静态外观和空间布局特征；时间流以连续帧之间的光流图或差分图像为输入，主要学习运动方向、速度和动态变化信息，最后再将两个分支的分类结果或特征进行融合。它的优势是能够显式利用视频中的空间信息和运动信息，两类特征互补，尤其适合既依赖外观又依赖动作变化的识别任务。其局限性也比较明显：光流预计算开销大、存储成本高；两个流通常独立处理，跨流信息交互不足；空间流偏静态，时间流依赖光流质量，在快速运动或复杂场景中可能不稳定；同时传统双流网络对长时间视频的建模能力有限，往往只能利用较短片段中的时序上下文。

## 3）三维卷积神经网络（3D CNN）如何解决传统二维卷积的不足？其代表模型有哪些？

答案：传统二维卷积神经网络主要在单帧图像的高度和宽度两个空间维度上提取特征，适合图像识别，但难以直接建模视频帧之间的时间变化，因此对动作中的连续运动信息捕捉不足。3D CNN 将卷积核扩展到“空间 + 时间”三个维度，在视频序列上滑动三维卷积核，使网络能够同时提取空间结构和连续帧之间的动态变化，从而端到端地学习深层时空特征，减少对人工特征或光流预处理的依赖。PDF 中提到的代表模型包括 C3D，它将视频帧序列整体输入网络并用 3D 卷积提取时空特征；I3D，它将 2D 卷积核膨胀为 3D 卷积核，并结合 RGB 与光流双流结构；R3D，它把 ResNet 残差结构扩展到三维卷积以缓解深层网络训练问题；SlowFast，它通过慢速分支建模长时特征、快速分支捕捉短时快速变化，在精度和效率之间取得平衡。

## 4）循环神经网络（RNN）在动作识别中发挥什么作用？有哪些典型结构改进？

答案：RNN 在动作识别中的主要作用是对视频帧序列进行时间建模，捕捉连续帧之间的时序依赖关系和动作演变过程。常见做法是先使用 2D CNN 提取每一帧或每个片段的空间特征，再把这些帧级特征输入 RNN，以学习动作从开始到结束的动态变化。由于普通 RNN 在长序列中容易出现梯度消失或难以记住远距离信息的问题，典型改进包括 LSTM、Bi-LSTM 和 GRU。LSTM 通过门控机制建模长期依赖，适合理解较长时间范围内的动作变化；Bi-LSTM 同时利用前向和后向时序信息，增强上下文理解能力；GRU 结构更简洁、参数更少，在保持较好建模能力的同时提升计算效率。此外，一些方法还将注意力机制与 LSTM 结合，使模型更加关注关键帧和重要特征，从而提升动作识别的判别能力。

## 5）基于 Transformer 的方法相比 CNN/RNN 有何优势？其代表性模型有哪些创新点？

答案：基于 Transformer 的动作识别方法利用自注意力机制建模视频序列中任意位置之间的依赖关系，因此相比 CNN 更擅长捕捉全局关系和长距离时空依赖，相比 RNN 则不需要严格按时间顺序递归处理，能够并行计算，训练效率和建模灵活性更好。它可以通过空间注意力、时间注意力或时空联合注意力来处理复杂视频关系，在长序列动作识别中具有优势。PDF 中提到的代表模型包括将 ViT 扩展到视频任务的方法，它把视频切分为帧级补丁并引入分割注意力或时空注意力来捕捉视频特征；RegionViT 等工作侧重降低注意力计算带来的复杂度和内存开销；VideoMAE 和 VideoMAEv2 则借鉴掩码自编码思想，通过遮蔽部分视频帧或视频块并让模型预测被遮蔽内容，在较少标注甚至自监督条件下学习有效的时空表示，从而缓解视频标注成本高的问题。

## 6）点云数据常用的获取方式有哪些？

点云数据通常由能够获取三维空间信息的设备或方法生成，常见方式包括深度相机、RGB-D 相机、激光雷达、结构光传感器、多视角相机重建以及由人体模型或仿真系统生成三维数据等。对于点云动作识别而言，传感器可以在连续时间上采集人体表面或场景中物体的三维坐标，形成点云序列，从而直接反映人体在三维空间中的姿态、形状和运动轨迹。与普通 RGB 视频只记录二维颜色和纹理不同，点云数据更强调空间坐标、几何结构和深度信息，因此对光照变化和背景纹理不那么敏感，在需要精确分析人体三维动作的场景中具有优势。

## 7）点云数据与视频数据在动作识别中的本质区别是什么？

答案：点云数据与 RGB 视频数据在动作识别中的本质区别在于数据表达形式和关注信息不同。RGB 视频由连续二维图像帧组成，主要包含颜色、纹理、外观、背景和二维视觉上下文，动作识别时通常需要从帧间变化中推断运动信息；点云数据则由三维空间中的无序点集组成，每个点通常表示空间坐标，有时还带有颜色、法向量等属性，它直接反映人体或物体在三维空间中的几何结构、空间关系和深度变化。因此，RGB 视频更依赖外观与场景线索，但容易受到光照、背景、视角和遮挡影响；点云数据更注重三维几何和运动轨迹，对光照和背景变化不敏感，但会面临点集无序、稀疏、不规则、点数变化和跨帧对应困难等问题。

## 8）点云动作识别的主要技术挑战有哪些？

答案：点云动作识别的主要技术挑战来自点云数据本身的无序性、不规则性、稀疏性以及时序建模难度。首先，点云不像图像那样具有规则网格结构，不能直接套用普通 2D CNN，需要设计能够处理无序点集的网络；其次，点云在空间上可能存在遮挡、缺失、噪声和采样密度不均，导致局部几何结构不稳定；再次，点云序列在时间上是离散的，不同帧之间缺少天然的一一对应关系，容易出现点追踪误差和时空关系紊乱；此外，体素化方法虽然可以把点云转成规则结构，但计算和存储开销较大，点跟踪方法又可能产生误差累积。对于半监督和自监督任务，点云标注成本高、无标注数据利用困难、辅助任务设计不稳定等问题也会限制模型性能。

## 9）不同监督范式的点云动作识别任务有什么区别和联系？

答案：点云动作识别按照监督信息的不同，可以分为全监督、半监督和自监督三类。全监督点云动作识别使用全部带有动作类别标签的点云序列进行训练，模型直接学习从点云时空特征到动作类别的映射，通常性能较好，但高度依赖人工标注数据；半监督点云动作识别只使用少量带标签数据和大量无标签数据，通过伪标签、掩码重构等方式利用无标签样本，目标是在降低标注成本的同时保持较好的识别效果；自监督点云动作识别完全不依赖人工动作标签，而是设计辅助任务，如片段顺序恢复、掩码时空结构预测、时序基数差预测或教师-学生蒸馏，让模型先自主学习点云序列的时空表示，再迁移到下游动作识别任务。三者的联系在于都需要学习点云动作的空间几何结构和时间动态变化，只是监督信号来源不同：全监督来自真实类别标签，半监督来自真实标签加伪标签或重构监督，自监督来自预设辅助任务。

## 10）阐述一下全监督点云动作识别任务中不同网络的优缺点。

答案：全监督点云动作识别中常见网络包括基于卷积 + Transformer 的方法、基于追踪 + PointNet/PointNet++ 的方法以及体素化相关方法。卷积 + Transformer 类方法如 P4-Transformer、PSTCN、PST-Transformer，能够结合局部时空卷积和全局注意力机制，既提取局部邻域特征，又建模长距离依赖，适合处理复杂点云序列；但 Transformer 通常计算复杂度较高，直接融合空间和时间维度还可能导致时空关系混乱，因此需要解耦或优化结构。基于 PointNet/PointNet++ 与追踪的方法能够直接处理原始点云，利用局部区域或 patch 轨迹描述人体运动，避免完全体素化带来的巨大开销，patch 追踪相比单点追踪容错性更强；但点追踪或区域构建仍可能受到噪声、遮挡和跨帧匹配误差影响。体素化方法如 3DV 可以把点云转换为规则体素结构，便于使用卷积网络提取运动和几何特征，但体素化过程计算资源消耗大，分辨率选择也会影响细节保留和效率。总体来看，不同网络在精度、计算成本、时空建模能力和对点云不规则性的适应性之间各有取舍。
